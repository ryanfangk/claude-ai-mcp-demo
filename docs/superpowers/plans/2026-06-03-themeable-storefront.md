# Themeable Storefront Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a themeable micro-storefront where one MCP-driven write to `SiteSettings` + `Pages.sections` + `Products` rebrands the visible site (palette, typography, copy, catalog) end-to-end. Drives the "Digital AI Sharing" demo moment.

**Architecture:** A `SiteSettings` global holds theme tokens (palette, typography, radius, density) + brand identity. A `Pages` collection holds composed `sections` (5 block types). The root `(frontend)/layout.tsx` reads `SiteSettings`, emits **CSS variables** on `<html>`, and loads fonts via `next/font/google`. A catch-all `[[...slug]]/page.tsx` route fetches a `Pages` row and maps `sections` → React components, each styled exclusively through the emitted vars. MCP writes land as drafts via the existing `forceDraftForMcpWrites` hook (for `Pages`) and via explicit `draft: true` in the `updateSiteSettings` tool handler (for the global).

**Tech Stack:** Next.js 16 (App Router) · React 19 · Payload CMS 3.85 · Postgres (Drizzle migrations) · `@payloadcms/plugin-mcp` · `next/font/google` · CSS Modules + plain `globals.css` (NO Tailwind).

**Spec:** [`docs/superpowers/specs/2026-06-03-themeable-storefront-design.md`](../specs/2026-06-03-themeable-storefront-design.md).

**Project commit/push rule:** per `.claude/rules/workflow.md` "Commit and push require explicit approval" — every task ends at **stage** (`git add`). Do NOT `git commit` or `git push`; the operator does that. Each task lists what to stage; the operator decides commit boundaries.

```tsx
// app/(frontend)/[[...slug]]/page.tsx
import { notFound } from 'next/navigation'
import config from '@payload-config'
import { getPayload } from 'payload'
import { renderSection } from '../_blocks/renderSection'

export const dynamic = 'force-dynamic'

export default async function PageRoute({
  params,
}: {
  params: Promise<{ slug?: string[] }>
}) {
  const { slug } = await params
  const slugStr = slug?.join('/') ?? 'home'
  const payload = await getPayload({ config })
  const { docs } = await payload.find({
    collection: 'pages',
    where: { slug: { equals: slugStr } },
    depth: 1,
    limit: 1,
    pagination: false,
  })
  const page = docs[0]
  if (!page) notFound()

  const sections = (page.sections ?? []) as Array<{ blockType: string }>
  return <main>{sections.map((s, i) => renderSection(s, i))}</main>
}
```

Add this helper above the `SiteSettings` export:

```ts
function paletteField(name: string, defaultValue: string, description: string) {
  return {
    name,
    type: 'text' as const,
    required: true,
    defaultValue,
    validate: (val: string | null | undefined) => {
      if (val == null) return true
      return /^#[0-9a-fA-F]{6}$/.test(val) ? true : `${name} must be a 6-digit hex color like #1c1917`
    },
    admin: { description },
  }
}
```

```ts
export type Palette = {
  bg: string
  surface: string
  text: string
  muted: string
  accent: string
  accentText: string
}

export type SiteSettingsForTheme = {
  palette: Palette
}

export function paletteToCssVars(p: Palette): Record<string, string> {
  return {
    '--bg': p.bg,
    '--surface': p.surface,
    '--text': p.text,
    '--muted': p.muted,
    '--accent': p.accent,
    '--accent-text': p.accentText,
  }
}
```

- [ ] **Step 4: Emit vars from the layout**

Emit the vars in a `<style>` tag in `<head>` as `:root { ... }`. (Don't use a React inline `style={{ ... }}` on `<html>` — CSS custom properties don't compose cleanly through React's CSSProperties type, and a `<style>` tag also lets us scope to `:root` so the vars cascade to every element.)

Modify `app/(frontend)/layout.tsx`:

```tsx
import type { Metadata } from 'next'
import config from '@payload-config'
import { getPayload } from 'payload'
import { paletteToCssVars, type Palette } from './_lib/theme'
import './globals.css'

export async function generateMetadata(): Promise<Metadata> {
  const payload = await getPayload({ config })
  const settings = await payload.findGlobal({ slug: 'site-settings' })
  return { title: settings.brandName ?? 'storefront' }
}

export default async function FrontendLayout({ children }: { children: React.ReactNode }) {
  const payload = await getPayload({ config })
  const settings = await payload.findGlobal({ slug: 'site-settings' })
  const cssVars = paletteToCssVars(settings.palette as Palette)
  const rootDecl = Object.entries(cssVars)
    .map(([k, v]) => `${k}: ${v};`)
    .join(' ')

  return (
    <html lang="en" suppressHydrationWarning>
      <head>
        <style>{`:root { ${rootDecl} }`}</style>
      </head>
      <body>{children}</body>
    </html>
  )
}
```

```ts
export const allFontVariables = `${inter.variable} ${lora.variable} ${playfair.variable} ${jetbrains.variable}`

export function fontFamilyVar(choice: Typography): string {
  switch (choice) {
    case 'sans': return 'var(--font-inter)'
    case 'serif': return 'var(--font-lora)'
    case 'display': return 'var(--font-playfair)'
    case 'mono': return 'var(--font-jetbrains)'
  }
}
```

```ts
import type { Radius, Density } from '@/lib/theme-tokens'

const RADIUS_PX: Record<Radius, number> = { sharp: 0, soft: 6, rounded: 14 }
const DENSITY_SCALE: Record<Density, number> = { tight: 0.85, standard: 1, airy: 1.2 }

export type SiteSettingsForTheme = {
  palette: Palette
  typography: { heading: Typography; body: Typography }
  radius: Radius
  density: Density
}

export function settingsToCssVars(s: SiteSettingsForTheme): Record<string, string> {
  const r = RADIUS_PX[s.radius]
  const d = DENSITY_SCALE[s.density]
  return {
    ...paletteToCssVars(s.palette),
    '--font-heading': fontFamilyVar(s.typography.heading),
    '--font-body': fontFamilyVar(s.typography.body),
    '--radius-sm': `${r / 2}px`,
    '--radius-md': `${r}px`,
    '--radius-lg': `${r * 2}px`,
    '--space-1': `${0.25 * d}rem`,
    '--space-2': `${0.5 * d}rem`,
    '--space-3': `${1 * d}rem`,
    '--space-4': `${1.5 * d}rem`,
    '--space-5': `${2 * d}rem`,
    '--space-6': `${3 * d}rem`,
  }
}
```

```tsx
// app/(frontend)/_blocks/FeaturedProductsSection.tsx
import config from '@payload-config'
import { getPayload } from 'payload'
import type { Where } from 'payload'
import styles from './FeaturedProductsSection.module.css'

export type FeaturedProductsSectionProps = {
  heading?: string | null
  limit: number
  category?: string | null
}

export async function FeaturedProductsSection(props: FeaturedProductsSectionProps) {
  const { heading, limit, category } = props
  const payload = await getPayload({ config })
  const where: Where = { _status: { equals: 'published' } }
  if (category && category !== 'all') {
    where.category = { equals: category }
  }
  const { docs } = await payload.find({
    collection: 'products',
    where,
    limit,
    depth: 0,
    sort: '-createdAt',
  })

  return (
    <section className={styles.section}>
      {heading ? <h2 className={styles.heading}>{heading}</h2> : null}
      <ul className={styles.grid}>
        {docs.map((p) => (
          <li key={p.id} className={styles.card}>
            <div className={styles.icon}>{p.icon ?? '📄'}</div>
            <h3 className={styles.title}>{p.title}</h3>
            {p.subtitle ? <p className={styles.subtitle}>{p.subtitle}</p> : null}
            <p className={styles.price}>
              {p.price === 0 ? 'free' : `${p.price.toFixed(2)} ${p.currency.toUpperCase()}`}
            </p>
          </li>
        ))}
      </ul>
    </section>
  )
}
```

```ts
const hexColor = z
  .string()
  .regex(/^#[0-9a-fA-F]{6}$/, 'must be a 6-digit hex color like #1c1917')

const siteSettingsPatchSchema = z
  .object({
    brandName: z.string().min(1).optional(),
    tagline: z.string().optional(),
    vibe: z.string().optional(),
    palette: z
      .object({
        bg: hexColor.optional(),
        surface: hexColor.optional(),
        text: hexColor.optional(),
        muted: hexColor.optional(),
        accent: hexColor.optional(),
        accentText: hexColor.optional(),
      })
      .optional()
      .describe('Theme palette. Any subset of fields supplied is merged into the existing palette.'),
    typography: z
      .object({
        heading: z.enum(TYPOGRAPHY_VALUES).optional(),
        body: z.enum(TYPOGRAPHY_VALUES).optional(),
      })
      .optional(),
    radius: z.enum(RADIUS_VALUES).optional(),
    density: z.enum(DENSITY_VALUES).optional(),
  })
  .refine(
    (d) => Object.keys(d).length > 0,
    { message: 'At least one field must be supplied.' },
  )

export const updateSiteSettingsTool: McpCustomTool = {
  name: 'updateSiteSettings',
  description:
    'Update the SiteSettings global (theme tokens + brand identity) as a DRAFT. Any subset of fields can be supplied: brandName, tagline, vibe, palette {bg, surface, text, muted, accent, accentText} (each #rrggbb hex), typography {heading, body} (sans|serif|display|mono), radius (sharp|soft|rounded), density (tight|standard|airy). Nested `palette` and `typography` patches are merged with current values — fields you omit keep their existing value. The site re-skins on the next page render (a human admin publishes to make changes public).',
  parameters: { patch: siteSettingsPatchSchema as z.ZodTypeAny },
  handler: async (args: Record<string, unknown>, req: PayloadRequest) => {
    const denied = authGuard(req, 'update site settings')
    if (denied) return denied

    const parsed = siteSettingsPatchSchema.safeParse(args.patch)
    if (!parsed.success) return parseError('patch', parsed.error)

    const patch = parsed.data
    const payload = req.payload

    // Merge nested groups so the agent doesn't need to know the full prior state
    // to update one palette field.
    const current = (await payload.findGlobal({ slug: 'site-settings' })) as {
      palette?: Record<string, string>
      typography?: { heading?: string; body?: string }
    }
    const merged: Record<string, unknown> = { ...patch }
    if (patch.palette) merged.palette = { ...(current.palette ?? {}), ...patch.palette }
    if (patch.typography) merged.typography = { ...(current.typography ?? {}), ...patch.typography }

    try {
      await payload.updateGlobal({
        slug: 'site-settings',
        data: merged,
        draft: true,
        overrideAccess: false,
        req,
      })
    } catch (err) {
      return {
        content: [
          {
            type: 'text' as const,
            text: `Error updating site-settings: ${err instanceof Error ? err.message : String(err)}`,
          },
        ],
      }
    }

    const summary = {
      status: 'draft',
      updated: Object.keys(patch),
      paletteMerged: !!patch.palette,
      typographyMerged: !!patch.typography,
    }
    return { content: [{ type: 'text' as const, text: JSON.stringify(summary, null, 2) }] }
  },
}
```

```tsx
import { headers as nextHeaders } from 'next/headers'

export default async function PageRoute({
  params,
}: {
  params: Promise<{ slug?: string[] }>
}) {
  const { slug } = await params
  const slugStr = slug?.join('/') ?? 'home'
  const payload = await getPayload({ config })

  const headers = await nextHeaders()
  const auth = await payload.auth({ headers })
  const isAdmin = auth.user?.collection === 'admins'

  const { docs } = await payload.find({
    collection: 'pages',
    where: { slug: { equals: slugStr } },
    depth: 1,
    limit: 1,
    pagination: false,
    draft: isAdmin,
    overrideAccess: !isAdmin, // admin path: respect access; public path: keep override (matches access fn already)
  })
  const page = docs[0]
  if (!page) notFound()

  const sections = (page.sections ?? []) as Array<{ blockType: string }>
  return <main>{sections.map((s, i) => renderSection(s, i))}</main>
}
```

## Self-review notes (for the implementer)

A few things to know that aren't in any single task:

- **`payload-types.ts`** is auto-generated; you may want to run `pnpm generate:types` after Task 1, Task 3, and again after Task 6 so the block/global types match. Not required for any task to pass, but reduces the `as unknown as` casts in renderSection.
- **Restart `pnpm dev` after each schema change.** Hot reload covers code, not Payload's collection/block registration on boot.
- **The workflow rule on commits:** every task ends at "stage". Do not commit. The operator decides the commit boundary — could be one big "feat: themeable storefront" or one commit per task. Their call.
- **Hand-edit migrations** when Payload's generator emits NOT NULL without a backfill (the migration-authoring rule). Add a `db.execute(sql\`UPDATE ... SET ... WHERE ... IS NULL\`)` before the `ALTER COLUMN ... SET NOT NULL`, or set a `DEFAULT` clause in `ADD COLUMN`.
